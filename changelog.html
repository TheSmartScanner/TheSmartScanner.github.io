SmartScanner Release Notes

Version 2.2 – Aug 1, 2025

🚀 New Features

  • Sensitive Data Disclosure Test: Added a new test for detecting Sensitive Data Disclosure, helping you identify exposed information like API keys, credentials, or internal paths.
  • No-Crawl Option: You can now disable crawling entirely when providing a predefined list of URLs—ideal for targeted scans.
  • JavaScript File Scanning: SmartScanner can now analyze standalone JavaScript files, expanding coverage for client-side vulnerabilities.

⚡ Improvements

  • User Experience: The manual login browser now auto-fills the target URL, saving time and improving usability during authenticated scans.

🔄 Updates

  • Detection Rule Enhancements: Updated vulnerability checks for:
    • Vulnerable Apache Version
    • Vulnerable Tomcat Versions
    • Vulnerable PHP Versions
    • Vulnerable OpenSSL Versions
  • Bug Fixes:
    • Fixed a crash that could occur during scans in rare cases.
    • Fixed an issue where HTTP responses were improperly truncated in reports, causing the proof of concept to be partially hidden.

Version 2.1 – May 19, 2025

🚀 New Features

  • Renamed Vulnerability: Remote URL Disclosure has been renamed to Server-Side Request Forgery (SSRF), now with a dedicated toggle in the test settings.
  • Multi-URL Input Support: You can now input multiple target URLs directly in the target field.
  • File-Based Scanning: Upload and scan a file containing a list of URLs.
  • Extended Payload Testing: Deeper vulnerability tests using a broader range of payloads for user-defined URLs.
  • Manual Crawl Scope: Define manual crawl scope in scan configurations.
  • JavaScript Evaluation Toggle: Option to disable Chromium-based JavaScript evaluation in scan settings.

⚡ Improvements

  • Enhanced Crawler Performance: Improved handling of AJAX-driven content and dynamic web pages.
  • Smarter URL Handling: Better recognition and support for SEO-friendly and complex URL structures.
  • Improved HTTP Response Detection: More accurate identification of custom 200 OK and 404 Not Found pages.

🔄 Updates

  • Detection Rule Enhancements: Updated vulnerability checks for:
    • Vulnerable Tomcat Versions
    • Vulnerable PHP Versions
    • Vulnerable WordPress Versions
  • Bug Fixes:
    • Resolved an issue with license registration in the Pro version.
    • General stability and compatibility improvements through backend updates.

Version 2.0 – February 15, 2025

New Features

  • Command-Line Interface (CLI): Introduced sms, a CLI version of SmartScanner for automation and scripting.
  • IDOR Detection: Added tests for Insecure Direct Object Reference (IDOR) vulnerabilities.
  • Fuzzing Tests: Enabled fuzzing in scan configurations for better vulnerability discovery.
  • Crawl Depth Control: Setting crawl depth to 0 now scans only the input URL without crawling additional pages.
  • Subdomain Scanning Control: Subdomains are no longer scanned by default, reducing unnecessary requests.

Improvements

  • Reduced False Positives: Enhanced passive tests to minimize incorrect vulnerability detections.
  • Performance Boost: Optimized scanning speed and resource usage.
  • Enhanced Crawling: Improved accuracy and efficiency in web crawling.
  • Manual Login Fixes: Resolved bugs affecting manual authentication workflows.
  • Stronger Input Validation: Improved input URL validation to prevent scanning errors.
  • UI Enhancements: Made subtle UI adjustments for a better user experience.

Updates

  • Vulnerability Detection Enhancements: Updated detection rules for:
    • Vulnerable OpenSSL Versions.
    • Vulnerable Nginx Versions.
    • Vulnerable Tomcat Versions.
  • System Requirements Update:
    • Dropped support for 32-bit systems.
    • Windows 10 versions older than 1809 are no longer supported.
  • Chromium Update: Upgraded embedded Chromium to version 122 for better compatibility and security.
